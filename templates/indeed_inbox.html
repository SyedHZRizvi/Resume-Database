How to set up the Indeed inbox

The app reads Indeed application notifications out of an IMAP-accessible mailbox (the same one Indeed sends notifications to). Set these environment variables on the host (Render → Settings → Environment), then redeploy.

VariableRequired?Example / Notes
INDEED_IMAP_HOST Yes imap.gmail.com / 127.0.0.1 for Proton Bridge
INDEED_IMAP_PORT No Defaults to 993 (SSL). Use 143 for STARTTLS / plain.
INDEED_IMAP_USER Yes Full mailbox login (usually the email address)
INDEED_IMAP_PASSWORD Yes App password (Gmail) or bridge password (Proton). For Gmail visit myaccount.google.com/apppasswords.
INDEED_IMAP_FOLDER No Defaults to INBOX
INDEED_IMAP_USE_SSL No Defaults to true
INDEED_SENDER_DOMAINS No Defaults to indeed.com,indeedemail.com
INDEED_CRON_TOKEN Yes Long random string — used to authenticate the cron URL. Generate one with openssl rand -hex 24.

Steps:

  1. Create a dedicated mailbox (or reuse the existing HR address) that Indeed sends application notifications to.
  2. Generate an app password for that mailbox (Gmail: My Account → Security → App Passwords; Proton: install Bridge and use the bridge password).
  3. On Render, open Settings → Environment and add the eight variables above. Save and redeploy.
  4. Return to this page — the status pill will switch to Configured .
  5. Copy the cron URL shown above into a free scheduler such as cron-job.org and set it to run every 5–15 minutes.

The password and cron token are never displayed on this page or in the audit log. Treat the cron URL as a secret — anyone who has it can trigger a poll.
